Lesson 1: Getting Started with HTML

How the web works

The web is a network of computers that exists as a collection of hubs. A computer will send an http request to a server, which then finds an appropraite html document and returns it to the user's computer thorugh a browser.

HTML = hyper text markup language

hyper - links
text - what you see
markup - how it looks
language - programming syntax

Tags and elements

html documents are composed of elements. We instruct the webpage to incorporate elements through the use of tags.

Inline vs block elements

html elements are either inline or block. Block elements form an invisible block around them, so will reflect more detail in text arrangements.

Lesson 2: Creating a Structured Document with HTML

Developer Tools (in the browser)

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on... In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

Text Editors (for programming)

When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Methods for including CSS Styling in web pages

Method 1: write CSS in the head of your HTML.
Method 2: Link your HTML to a separate CSS file
Method 3: Write your style inline with your HTML

Box Sizing

There are four main points that Jessica addressed about box sizing:
1. HTML elements are boxes and each box has 4 components
2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
3. There are two techniques you can use to help deal with sizing issues: a/ Set sizes in terms of percentages rather than pixels; b/ Set the box-sizing attribute to border-box for every element.
4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.

Box Positioning

Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.

Avoid Repetition

Avoidance of repetition in your code allows elements to be modified without further alterations to other logically unrelated elements. Additionally, all elements that are logically related will change predictably and uniformly, thus are kept in sync.
